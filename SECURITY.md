<h2 id="reporting-a-vulnerability" align="center">Reporting a Vulnerability</h2>

<p>
  If you discover a security issue in <code>x.sh</code>, scripts under <code>scripts/</code>, or setup files under <code>wsl/</code>,
  please report it privately by email:
</p>

<p align="center">
  <a href="mailto:x@xscriptor.com"><b>x@xscriptor.com</b></a>
</p>

<blockquote>
  Please do not open public issues for vulnerabilities until a fix is available.
</blockquote>

<h2 id="what-to-include" align="center">What to Include in a Report</h2>

<ul>
  <li>Affected script/file and commit or branch.</li>
  <li>Clear reproduction steps and expected vs. actual behavior.</li>
  <li>Impact analysis (privilege escalation, command injection, data exposure, etc.).</li>
  <li>Any logs or proof-of-concept details that help verification.</li>
</ul>
